# Security Policy

## Supported Versions

The active development branch is `develop`. Security fixes should be applied there first and then merged through the normal release flow.

## Reporting a Vulnerability

If you discover a security issue, please do not open a public issue describing the vulnerability.

Instead, report it using one of the following methods:

- Open a private security report if your Git hosting setup supports it
- Contact the maintainers through the project’s official channel

Include as much detail as possible:

- A short summary of the issue
- Steps to reproduce
- Affected files or features
- Any proof of concept, logs, or screenshots

## What to Expect

We will review the report, confirm the issue, and coordinate a fix before broader disclosure when appropriate.
