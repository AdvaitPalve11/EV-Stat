# Contributing to FuelPay

Thank you for helping improve FuelPay.

## Workflow

FuelPay follows a branch-based workflow:

1. Branch from `develop`
2. Keep work focused on one feature, fix, or phase
3. Use short, descriptive commit messages
4. Open a pull request back into `develop`
5. Merge only after review and validation

## Commit Style

Use messages that describe the change clearly, for example:

- `feat: add premium loading state`
- `fix: correct payment router path`
- `docs: update privacy policy`

## Pull Requests

Each pull request should include:

- A short description of the change
- Screenshots or recordings for UI work when relevant
- Notes about validation performed
- Any follow-up work that remains

## Code Quality

- Keep changes focused and small when possible
- Match the existing Flutter and Dart style in the repo
- Add tests for new behavior when practical
- Avoid unrelated refactors in the same pull request

## Reporting Issues

When filing an issue, include:

- A clear summary
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots, logs, or stack traces if available
